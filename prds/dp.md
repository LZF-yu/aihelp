1. 文档概述

1.1 文档目的

本文档明确AI智能待办助手项目的核心需求、功能范围、技术要求、界面规范及验收标准，作为项目开发、测试、验收的唯一依据，确保开发过程贴合需求，最终交付可落地、可使用的AI驱动型待办管理系统，适配应届生开发能力，兼顾实用性与AI亮点，可作为毕业设计或简历项目使用。

1.2 项目背景

传统待办工具仅能实现任务的增删改查，无法满足用户对任务规划、优先级排序、高效管理的核心需求。随着大模型技术的普及，AI赋能个人效率工具成为趋势，本项目基于软件工程技术栈，结合大模型API，开发一款“传统待办+AI增强”的智能助手，解决用户任务拆解、计划生成、进度总结等痛点，同时体现AI应用落地能力，适配应届生项目开发场景。

1.3 项目目标

- 功能目标：实现待办任务基础管理，叠加AI智能拆解、优先级排序、计划生成、进度总结等核心AI功能，满足个人日常任务管理需求。

- 技术目标：基于SpringBoot+Vue3实现前后端分离架构，熟练接入大模型API，完成数据库设计、接口开发、前端交互，掌握AI应用层开发流程。

- 交付目标：完成可运行的Web版本，包含完整功能模块，无明显Bug，可直接用于毕业设计答辩或简历展示，同时提供可扩展空间。

- 体验目标：界面简洁易用，操作流畅，AI响应速度快（≤3秒），交互逻辑清晰，符合普通用户使用习惯。

1.4 适用范围

本文档适用于本项目的开发人员、测试人员、设计人员及验收人员，明确各角色的工作依据；同时适配应届生开发场景，所有技术选型、功能设计均控制在软件工程专业应届生能力范围内，无需掌握复杂算法或模型训练知识。

定义

AI增强功能

2. 总体需求

2.1 产品定位

一款面向个人用户的AI智能待办助手，以“基础待办+AI增强”为核心，主打“简单、高效、智能”，帮助用户快速拆解任务、规划时间、跟踪进度，同时作为应届生AI应用开发项目，突出技术落地能力，兼顾实用性与展示性。

2.2 核心功能范围

本项目分为两大模块：基础待办模块、AI增强模块，所有功能均控制在应届生开发能力范围内，无需复杂技术实现，具体范围如下：

- 基础待办模块：用户登录/注册、待办任务CRUD、任务分类、任务状态管理、仪表盘展示。

- AI增强模块：任务自动拆解、智能优先级排序、每日/每周计划生成、任务进度总结、提醒话术优化。

2.3 技术选型要求

技术栈需贴合软件工程专业应届生所学知识，优先选择主流、易上手、有丰富资料的技术，具体要求如下：

- 前端：Vue3 + Element Plus（或Vant），实现页面展示与交互，无需复杂前端框架。

- 后端：SpringBoot，负责接口开发、数据处理、大模型API接入，简化开发流程。

- 数据库：MySQL，存储用户信息、待办任务、任务分类等数据，表结构简单易懂。

- AI接口：优先选择免费额度充足、接入简单的第三方大模型API（通义千问、文心一言、讯飞星火），无需本地部署模型。

- 可选技术：Redis（缓存历史对话、常用AI响应）、JWT（用户登录认证），用于提升项目亮点。

2.4 性能要求

- 响应速度：基础操作（增删改查）响应时间≤1秒，AI功能响应时间≤3秒（受网络及API速度影响）。

- 稳定性：连续运行24小时无崩溃，无数据丢失；多用户同时操作（≤10人）无卡顿。

- 兼容性：支持主流浏览器（Chrome、Edge、Firefox），适配电脑端（无需适配移动端，可可选扩展）。

2.5 安全要求

- 用户认证：采用账号密码登录，密码加密存储（MD5加密），防止明文泄露。

- 数据安全：用户数据、待办任务仅本人可见，禁止跨用户访问；数据库定期备份（可选）。

- 接口安全：接口调用需验证用户身份（JWT可选），防止非法调用；AI接口密钥妥善存储，不暴露在前端。

3. 详细功能需求

3.1 用户模块

3.1.1 注册功能

- 输入项：用户名（唯一）、密码（≥6位）、确认密码、邮箱（可选，用于找回密码）。

- 校验规则：用户名不可重复，密码与确认密码一致，邮箱格式正确（若填写）。

- 反馈：注册成功跳转至登录页，注册失败提示具体原因（如用户名已存在、密码不一致）。

3.1.2 登录功能

- 输入项：用户名、密码。

- 校验规则：用户名、密码匹配，不存在则提示“账号或密码错误”。

- 功能：登录成功跳转至首页（待办列表），支持“记住密码”（可选，本地存储），登录状态有效期7天。

3.1.3 个人中心

- 功能：查看个人信息（用户名、邮箱），修改密码、修改个人资料（可选）。

- 退出登录：点击退出，清除登录状态，跳转至登录页。

3.2 基础待办模块

3.2.1 待办任务添加

- 输入项：任务标题（必填）、任务描述（可选）、截止时间（可选）、任务分类（可选）、优先级（高/中/低，默认中）。

- 功能：点击“添加”按钮，保存任务至数据库，状态默认为“未完成”；支持快速添加（仅输入标题）和详细添加（填写所有项）。

- 反馈：添加成功提示“添加成功”，失败提示具体原因（如标题为空）。

3.2.2 待办任务查询与展示

- 展示方式：默认按“优先级+截止时间”排序，展示任务标题、截止时间、分类、优先级、状态。

- 筛选功能：支持按状态（未完成/已完成）、分类、优先级筛选，输入关键词搜索任务标题/描述。

- 详情查看：点击任务，展开查看任务详细信息（描述、创建时间、截止时间等）。

3.2.3 待办任务更新

- 功能：支持修改任务标题、描述、截止时间、分类、优先级，支持修改任务状态（未完成→已完成，已完成→未完成）。

- 操作：点击“编辑”按钮，弹出编辑窗口，修改后保存；点击任务状态切换按钮（如复选框），直接修改状态。

3.2.4 待办任务删除

- 功能：支持单个任务删除、批量删除（勾选多个任务，点击“批量删除”）。

- 反馈：删除前提示“确认删除？删除后不可恢复”，删除成功提示“删除成功”。

3.2.5 任务分类管理

- 功能：支持添加分类（如“学习”“工作”“生活”）、修改分类名称、删除分类（删除分类时，该分类下的任务自动归为“未分类”）。

- 展示：分类列表展示，点击分类，快速筛选该分类下的所有任务。

3.2.6 仪表盘展示

- 功能：展示今日待办数量、未完成任务总数、已完成任务总数、各分类任务数量统计（饼图/柱状图，可选）。

- 亮点：展示“今日重点任务”（AI推荐的高优先级、近截止时间任务）。

3.3 AI增强模块（核心亮点）

3.3.1 AI任务自动拆解

- 触发方式：用户添加任务时，点击“AI拆解”按钮；或对已存在的任务，点击“拆解任务”。

- 功能：用户输入复杂任务（如“完成毕业设计”），AI自动拆解为多个子任务，子任务包含标题、建议截止时间、建议优先级。

- 交互：拆解完成后，展示子任务列表，用户可修改子任务信息，确认后批量添加至待办任务。

- 技术实现：调用大模型API，传入提示词（如“请将任务【XXX】拆解为具体可执行的子任务，每个子任务包含标题、建议截止时间、优先级，格式清晰”），解析API返回结果，展示给用户。

3.3.2 AI智能优先级排序

- 触发方式：点击“AI排序”按钮，对当前筛选的待办任务进行优先级重新排序。

- 功能：AI根据任务的截止时间、重要性、难度，自动调整任务的优先级（高/中/低），并给出排序理由（可选）。

- 交互：排序完成后，刷新任务列表，按新的优先级展示；用户可手动调整优先级，覆盖AI排序结果。

3.3.3 AI计划生成

- 功能：支持生成每日计划、每周计划。

- 每日计划：用户点击“生成今日计划”，AI根据当前未完成任务、用户习惯（可选，如“上午学习、下午开发”），生成今日待办列表，分配具体时间段（如“9:00-10:00 完成任务A”）。

- 每周计划：用户输入每周目标（如“本周完成Java学习”），AI生成每周详细计划，拆分到每日，添加至待办任务。

- 交互：计划生成后，用户可编辑、删除计划中的任务，确认后批量添加至待办。

3.3.4 AI任务进度总结

- 触发方式：点击“生成日报”“生成周报”按钮，自动总结当日/当周任务完成情况。

- 功能：AI统计当日/当周已完成任务、未完成任务、未完成原因（可选），生成结构化总结文本，支持复制、导出（可选，导出为txt）。

- 示例：“今日共完成3项任务，未完成2项（任务A、任务B），建议明日优先完成任务A（截止时间明日），合理分配时间。”

3.3.5 AI提醒话术优化

- 功能：用户设置任务提醒（可选，基于截止时间），AI将生硬的提醒话术（如“任务A即将截止”）优化为人性化提示（如“温馨提醒：任务A的截止时间还有1小时哦，尽快完成吧～”）。

- 交互：用户可选择提醒风格（温柔、简洁、严肃），AI根据风格生成对应话术；提醒方式为页面弹窗（无需实现邮件/短信提醒，降低难度）。

3.4 辅助功能

- 任务导出：支持将待办任务导出为Excel/txt格式（可选，提升项目亮点）。

- 历史记录：查看任务操作历史（如添加、修改、删除时间）（可选）。

- 主题切换：支持浅色/深色主题切换（可选，提升用户体验）。

- 简洁明了：界面布局清晰，无冗余元素，重点突出待办任务和AI功能按钮。

- 操作便捷：常用功能（添加任务、AI拆解、筛选）按钮位置明显，操作步骤不超过3步。

- 风格统一：字体、颜色、按钮样式统一，符合现代Web应用风格，无需复杂设计（可复用Element Plus组件）。

4.2 核心页面布局

4.2.1 登录/注册页

- 布局：居中展示登录/注册表单，包含输入框、按钮、跳转链接（登录→注册，注册→登录）。

- 元素：用户名输入框、密码输入框、确认密码输入框（注册页）、登录/注册按钮、“忘记密码”链接（可选）。

4.2.2 首页（待办列表页）

- 顶部：导航栏（logo、个人中心、退出登录）、搜索框、添加任务按钮、AI功能按钮（AI拆解、AI排序、生成计划）。

- 左侧：分类列表、筛选条件（状态、优先级）。

- 中间：待办任务列表（展示任务核心信息、操作按钮）。

- 右侧：仪表盘（任务统计、今日重点任务）（可选，可放在顶部下方）。

4.2.3 任务编辑/添加页

- 布局：弹窗形式，包含输入框、下拉框、按钮，简洁直观。

- 元素：任务标题输入框、任务描述文本域、截止时间选择器、分类下拉框、优先级下拉框、AI拆解按钮、保存/取消按钮。

4.2.4 个人中心页

- 布局：左侧个人信息展示，右侧修改表单。

- 元素：用户名、邮箱展示，密码修改表单、个人资料修改表单（可选）、退出登录按钮。

5. 技术需求

5.1 后端技术需求

- 框架：SpringBoot 2.x/3.x（选择你熟悉的版本）。

- 接口设计：RESTful风格接口，返回JSON格式数据，包含状态码、提示信息、数据。

- 数据库：MySQL 8.0，表结构清晰，关联合理（用户表、任务表、分类表）。

- AI接口接入：封装大模型API调用工具类，处理API请求与响应，异常捕获（如API调用失败、网络异常）。

- 认证授权：可选JWT实现用户认证，拦截未登录请求。

- 异常处理：全局异常捕获，统一返回错误信息，避免程序崩溃。

5.2 前端技术需求

- 框架：Vue3 + Vite（或Vue CLI），使用Composition API。

- UI组件：Element Plus（电脑端），复用组件（如按钮、输入框、弹窗、表格），减少开发量。

- 交互：使用Axios调用后端接口，处理加载状态、错误提示；实现任务列表刷新、筛选、分页（可选）。

- AI功能交互：实现AI拆解、排序等功能的弹窗展示，加载动画（如旋转图标），提升用户体验。

- 本地存储：可选localStorage存储用户登录状态、记住密码等信息。

6. 验收标准

6.1 功能验收

6.1.1 用户模块

- 注册：用户名唯一，密码加密存储，注册成功跳转登录页，错误提示准确。

- 登录：账号密码匹配可正常登录，不匹配提示错误，登录状态有效。

- 个人中心：可查看、修改个人信息，退出登录功能正常。

6.1.2 基础待办模块

- 添加任务：必填项校验正常，添加成功后任务展示在列表中。

- 查询与筛选：可按状态、分类、优先级筛选，搜索功能准确，列表展示正常。

- 更新任务：可修改任务所有信息，状态切换正常，修改后即时刷新。

- 删除任务：单个、批量删除功能正常，删除前有确认提示。

- 分类管理：添加、修改、删除分类功能正常，分类关联任务正确。

- 仪表盘：任务统计准确，展示清晰。

6.1.3 AI增强模块

- AI任务拆解：输入复杂任务，可成功拆解为子任务，子任务信息合理，可批量添加。

- AI排序：点击排序按钮，可自动调整任务优先级，排序结果合理。

- 计划生成：可生成每日/每周计划，计划内容贴合用户需求，可编辑添加。

- 进度总结：可生成日报/周报，统计准确，总结文本清晰。

- 提醒话术优化：可生成不同风格的提醒话术，话术人性化。

6.2 性能验收

- 响应速度：基础操作≤1秒，AI功能≤3秒，无明显卡顿。

- 稳定性：连续运行24小时无崩溃，数据无丢失；多用户操作无异常。

- 兼容性：在主流浏览器中可正常运行，界面无错乱。

6.3 安全验收

- 密码加密：用户密码存储为加密形式，不泄露明文。

- 数据隔离：不同用户的任务、信息相互隔离，无法跨用户访问。

- 接口安全：未登录用户无法访问核心接口，AI接口密钥不暴露。

6.4 界面验收

- 布局清晰：各页面布局合理，功能按钮位置明显，操作便捷。

- 风格统一：字体、颜色、按钮样式统一，无明显视觉错乱。

- 交互友好：有明确的操作反馈（如成功提示、错误提示），加载状态有提示。

8. 风险与应对措施

潜在风险

应对措施

大模型API调用失败/额度不足

优先选择免费额度充足的API（如通义千问、讯飞星火），提前注册并预留额度；添加异常处理，API调用失败时提示用户“服务暂时不可用”。

前后端交互异常

后端接口开发完成后，先通过Postman测试接口；前端调用接口时，统一处理响应状态码，添加错误提示，避免程序崩溃。

功能过于复杂，无法按时完成

优先完成基础待办模块和核心AI功能（任务拆解、优先级排序），辅助功能（导出、主题切换）可作为可选功能，后期扩展。

数据库设计不合理，导致数据异常

严格按照需求文档中的表结构设计，添加外键关联，测试数据插入、修改、删除操作，确保数据一致性。
